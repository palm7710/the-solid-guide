## NOTE

### SOLID原則とは

- S：単一責任の原則
  - クラスはたった一つのアクターに対して責務を行うべき
- O：オープンクローズドの原則
- L：リスコフの置換原則
- I：インターフェース分離の原則
- D：依存性逆転の原則

### SOLID原則を学ぶ意義

- [x] コードの修正が想定外の箇所に影響してバグが生まれてしまう
- [x] 簡単な機能の追加で大量の既存コードを修正する必要がある
- [x] コードの依存関係が複雑で、理解するのに膨大な時間がかかる
- [x] 既存のコードが再利用しにくく、同じようなコードが量産される
- [x] コード間の依存度が強く、単体テストができない

### （補足）DRYの原則

- コードの繰り返しを避けろという内容の原則
- 同じようなロジックであっても、概念が違うものはDRYにすべきではない
- ビジネスを理解し、概念を理解していないと単一責任の原則を完全に満たすことは難しい
